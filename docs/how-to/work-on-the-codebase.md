# How to work on the codebase

This guide is for contributors (human or agent) changing apidef itself. The
single most important rule:

> **TypeScript (`ts/`) is canonical. Change it first, then bring Go (`go/`)
> into parity.** See
> [The canonical build and the parity port](../explanation/canonical-and-parity.md).

## The loop

```sh
# TypeScript: build (-> ts/dist, ts/dist-test) and test
npm run build
npm test

# Go: build and test the parity port
cd go && go build ./... && go test ./...

# Everything (TS + Go) in one go
make all
```

`npm test` runs the compiled tests under `ts/dist-test/`. To run a subset:

```sh
TEST_PATTERN='depluralize' npm run test-some
```

## Where things live

| area | path |
|------|------|
| pipeline entry | `ts/src/apidef.ts` |
| parse + `$ref` resolve | `ts/src/parse.ts` |
| classification | `ts/src/guide/{guide,heuristic01}.ts` |
| transforms (9 passes) | `ts/src/transform/*.ts` |
| builders | `ts/src/builder/**` |
| pure helpers | `ts/src/utility.ts` |
| types / model shapes | `ts/src/{types,model,desc,def}.ts` |
| Go port | `go/*.go` (flat package) |
| shared fixtures | `ts/test/*.tsv` |
| golden snapshots | `ts/test/model-ref/*.json` |

## When you change behavior

1. **Edit the TypeScript** in `ts/src/…` and add/extend a test.
2. **Update shared fixtures** in `ts/test/*.tsv` if a pure-function behavior
   changed — both languages execute these (`ts/test/tsv.test.ts` and
   `go/tsv_test.go`).
3. **Mirror into Go.** Find the matching code via the `// Mirrors src/...`
   comments, make the same change, and keep those comments accurate.
4. **Rebuild and commit `ts/dist` + `ts/dist-test`.** They are committed
   artifacts, regenerated by `npm run build`.
5. `make all` must be green.

## Adding a test

- Pure functions: add a row to the relevant `ts/test/*.tsv` (covers TS *and*
  Go automatically).
- TS unit/integration: a `*.test.ts` under `ts/test/` (mirror the
  `makeCtx`-style pattern in `ts/test/transform/*.test.ts`).
- Go: a `*_test.go` in `go/`.

## CI

`.github/workflows/build.yml` runs `npm ci && npm run build && npm test` on
Node 24.x and latest across Linux/macOS/Windows. Local Node may be older; the
`shape` peer dependency wants Node ≥ 24.

The Go golden comparisons (`go/validate_test.go`) read an external
`../../apidef-validate` checkout and `t.Skip` when it is absent, so a plain
`go test ./...` stays green without it.
